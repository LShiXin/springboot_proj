## 部署场景分析

### 场景1：部署到与MySQL、Redis相同的服务器（本地部署）
如果MySQL和Redis都安装在同一台服务器上，且应用也部署在这台服务器上：

**推荐配置：**
```properties
# MySQL配置 - 使用localhost或127.0.0.1
spring.datasource.url=jdbc:mysql://localhost:3306/project_dev?useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
# 或者使用127.0.0.1
# spring.datasource.url=jdbc:mysql://127.0.0.1:3306/project_dev?useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true

spring.datasource.username=remote_user  # 根据实际数据库用户修改
spring.datasource.password=Lzp1204@     # 根据实际数据库密码修改

# Redis配置 - 使用localhost或127.0.0.1
spring.data.redis.host=localhost
# 或者使用127.0.0.1
# spring.data.redis.host=127.0.0.1
spring.data.redis.port=6379
spring.data.redis.password=Lzp1204@     # 根据实际Redis密码修改
```

### 场景2：MySQL和Redis在同一内网，应用部署在另一台服务器
如果MySQL和Redis在同一内网服务器（例如192.168.1.100），应用部署在另一台服务器：

**推荐配置：**
```properties
# MySQL配置 - 使用内网IP
spring.datasource.url=jdbc:mysql://192.168.1.100:3306/project_dev?useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
spring.datasource.username=remote_user
spring.datasource.password=Lzp1204@

# Redis配置 - 使用内网IP
spring.data.redis.host=192.168.1.100
spring.data.redis.port=6379
spring.data.redis.password=Lzp1204@
```

### 场景3：生产环境部署（使用环境变量）
对于生产环境，建议使用环境变量或配置文件外部化：

**推荐配置：**
```properties
# MySQL配置 - 使用环境变量
spring.datasource.url=${DB_URL:jdbc:mysql://localhost:3306/project_dev?useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true}
spring.datasource.username=${DB_USERNAME:remote_user}
spring.datasource.password=${DB_PASSWORD:Lzp1204@}

# Redis配置 - 使用环境变量
spring.data.redis.host=${REDIS_HOST:localhost}
spring.data.redis.port=${REDIS_PORT:6379}
spring.data.redis.password=${REDIS_PASSWORD:Lzp1204@}
```

## 安全建议

### 1. 密码安全
- **不要**在代码仓库中硬编码生产环境密码
- 使用环境变量或配置管理工具（如Vault、AWS Secrets Manager）
- 为不同环境使用不同的密码

### 2. 网络访问控制
- 生产环境MySQL和Redis应该只允许特定IP访问
- 使用防火墙规则限制访问
- 考虑使用VPC或私有网络

### 3. SSL/TLS加密
- 生产环境建议启用MySQL SSL连接
- Redis可以考虑使用SSL/TLS（如果支持）

## 故障排除

### 常见问题1：连接被拒绝
```
java.net.ConnectException: Connection refused
```
**解决方案：**
1. 检查MySQL/Redis服务是否正在运行
2. 检查防火墙是否阻止了连接
3. 验证IP地址和端口是否正确

### 常见问题2：认证失败
```
Access denied for user 'remote_user'@'xxx.xxx.xxx.xxx'
```
**解决方案：**
1. 验证用户名和密码
2. 检查用户是否有从该IP地址连接的权限
3. 可能需要创建用户或授权：`GRANT ALL PRIVILEGES ON project_dev.* TO 'remote_user'@'%' IDENTIFIED BY 'password';`

### 常见问题3：数据库不存在
```
Unknown database 'project_dev'
```
**解决方案：**
1. 创建数据库：`CREATE DATABASE project_dev;`
2. 或者修改配置使用已存在的数据库
