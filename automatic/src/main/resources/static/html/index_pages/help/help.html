我是怎么工作的？

我的工作流程就像一个勤劳的小蜜蜂🐝：

  1. ⏰ 定时巡逻：我每10秒就会检查一次数据库，看看有没有到时间的监控任务
    @Scheduled(fixedRate = 10000) // 就是这里让我保持勤劳！
  2. 🔍 任务发现：找到到期的任务后，我会用"乐观锁"技术（防止重复干活）来安全地执行
    🎯 专业解释：乐观锁就像在多人协作时做个标记，确保我不会和别人抢着干同一件事！
  3. 🕷️ 爬虫出动：根据任务配置，我会派出对应的爬虫小分队：
    • 🐻 北注协小分队：负责爬取 beizhuxie.org.cn 的培训通知
    • 🐉 中注协小分队：负责爬取 cicpa.org.cn 的通知公告和要闻
  4. 📝 通知处理：爬取到通知后，我会：
    • 🎨 用关键词高亮显示重要内容
    • 💾 保存到数据库（不会重复保存哦）
    • 📊 记录执行情况，方便你查看统计
  5. 🔄 下次计划：根据你的设置，我会自动计算下次执行时间：
    • ⏱️ 固定频率：每隔X毫秒执行一次
    • ⏰ 固定延迟：上次任务完成后X毫秒执行
    • 📅 Cron表达式：像Linux定时任务一样精确控制
    • 🎯 一次性任务：只执行一次就收工

核心建议：一个任务，一个关键词！

这是我的超级重要的建议！让我告诉你为什么：

更精准

每个任务专注于一个关键词，匹配结果更准确，不会因为多个关键词相互干扰。

更好统计

执行记录和通知统计会更清晰，你知道每个关键词具体带来了多少新通知。

更易管理

如果某个关键词的通知突然增多，你可以快速定位是哪个关键词触发的。

更高性能

关键词越少，我的处理速度越快，可以更快地为你发现新通知！

📝 举个例子：

❌ 不推荐的做法：

培训,CPA,考试,报名,资格

✅ 推荐的做法：

任务1：培训
任务2：CPA
任务3：考试
任务4：报名

这样每个任务都很专注，管理起来超方便！

其他小贴士：

📅 合理设置时间：根据通知发布频率设置执行间隔，太频繁可能会给网站带来压力哦～

🔔 及时查看通知：我找到新通知后会在统计页面显示，记得经常来看看！

🛠️ 任务管理：不需要的任务记得及时关闭或删除，让我专心做重要的事！

🔍 关键词优化：使用更具体的关键词，比如"北京培训"比"培训"更精准！

遇到问题怎么办？

任务没有执行？

  • ✅ 检查任务状态是否为"激活"
  • ✅ 确认当前时间在任务的开始和结束时间范围内
  • ✅ 查看"执行记录"页面，看看是否有错误信息

没有收到通知？

  • ✅ 检查网站是否有新通知发布
  • ✅ 确认关键词设置是否正确
  • ✅ 查看通知是否在"最近一个月"的时间范围内

执行速度慢？

  • ✅ 减少每个任务的关键词数量
  • ✅ 适当增加执行间隔时间
  • ✅ 检查网络连接是否正常
