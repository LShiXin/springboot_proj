# 自动化监控管理系统 v1.1 - 项目介绍

## 项目概述

自动化监控管理系统是一个基于Spring Boot的Web应用程序，旨在提供智能化的网站监控、数据爬取和定时任务调度服务。系统通过集成多种爬虫工具，实现对目标网站的自动化监控、内容更新检测和实时通知功能。

## 核心功能

### 1. 智能爬虫监控系统
- **多源数据采集**：支持中国注册会计师协会(CICPA)、北京注协培训网等多个专业网站的数据爬取
- **实时监控**：对监控任务中的URL进行定期检查，检测内容更新
- **关键词高亮**：自动识别并高亮显示监控的关键词
- **智能去重**：避免重复采集相同内容，提高数据质量

### 2. 定时任务调度引擎
- **灵活调度**：支持CRON表达式、固定频率、固定延迟和一次性任务等多种调度类型
- **自动扫描**：每5分钟自动扫描并加载符合条件的定时任务
- **执行队列管理**：按执行时间排序创建任务执行队列
- **状态跟踪**：实时记录任务执行状态和统计信息

### 3. 监控任务管理
- **任务配置**：支持创建、编辑、删除监控任务
- **子链接管理**：支持对监控任务下的单个链接进行删除和状态切换
- **权限控制**：确保用户只能操作自己创建的任务和链接
- **数据一致性**：使用Redis缓存确保前后端数据同步

### 4. 通知与告警系统
- **实时通知**：当监控到内容更新时，系统自动生成通知
- **通知记录**：完整的通知历史记录和查看功能
- **用户友好界面**：清晰的通知展示和管理界面

### 5. 用户与权限管理
- **用户认证**：基于JWT的Token认证机制
- **权限验证**：拦截器验证用户权限和Token有效性
- **个性化统计**：按用户统计任务执行情况

## 技术架构

### 后端技术栈
- **框架**：Spring Boot 3.2.2
- **Java版本**：Java 17
- **数据库**：
  - MySQL 8.0.33（主数据存储）
  - Redis（缓存和会话管理）
- **数据访问**：Spring Data JPA
- **安全认证**：JWT (JSON Web Token)
- **HTTP客户端**：Apache HttpClient 4.5.14
- **HTML解析**：Jsoup 1.18.1
- **JSON处理**：Fastjson 1.2.83
- **日志框架**：SLF4J + Logback
- **构建工具**：Maven

### 前端技术栈
- **核心框架**：原生JavaScript + HTML5 + CSS3
- **UI组件**：自定义CSS样式组件
- **异步通信**：Fetch API
- **状态管理**：基于LocalStorage的客户端状态管理

### 系统架构特点
1. **分层架构**：清晰的Controller-Service-Repository分层设计
2. **异步处理**：使用@Async注解实现异步任务执行
3. **定时调度**：基于@Scheduled注解的定时任务框架
4. **事务管理**：使用@Transactional确保数据一致性
5. **异常处理**：统一的异常处理机制和错误响应

## 主要API接口

### 爬虫管理
- `POST /api/crawler/execute` - 执行爬虫任务
- `GET /api/crawler/tasks` - 获取所有监控任务
- `POST /api/crawler/tasks` - 创建监控任务
- `PUT /api/crawler/tasks/{id}` - 更新监控任务
- `DELETE /api/crawler/tasks/{id}` - 删除监控任务

### 监控任务管理
- `POST /api/monitottask/deleteLink` - 删除监控任务子链接
- `POST /api/monitottask/toggleLinkStatus` - 切换链接启用/禁用状态

### 系统状态
- `GET /api/home` - 首页欢迎信息
- `GET /api/service/status` - 获取服务状态
- `GET /api/task-execution/stats` - 获取系统任务执行统计
- `GET /api/task-execution/user-stats` - 获取用户任务执行统计

### 定时任务调度
- `POST /api/scheduling/start` - 启动调度服务
- `POST /api/scheduling/stop` - 停止调度服务
- `POST /api/scheduling/scan` - 手动触发扫描
- `POST /api/scheduling/status` - 检查服务状态

## 部署与运行

### 环境要求
- Java 17 或更高版本
- MySQL 8.0+
- Redis 6.0+
- Maven 3.6+

### 配置文件说明
项目使用 `src/main/resources/application.properties` 作为配置文件。部署时需要根据实际环境修改以下关键配置：

#### 1. MySQL数据库连接
```properties
# 当前配置（远程服务器）
spring.datasource.url=jdbc:mysql://47.95.30.65:3306/project_dev?useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
spring.datasource.username=remote_user
spring.datasource.password=Lzp1204@

# 如果部署到与MySQL相同的服务器，建议修改为：
spring.datasource.url=jdbc:mysql://localhost:3306/project_dev?useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
```

#### 2. Redis连接配置
```properties
# 当前配置（远程服务器）
spring.data.redis.host=47.95.30.65
spring.data.redis.port=6379
spring.data.redis.password=Lzp1204@

# 如果部署到与Redis相同的服务器，建议修改为：
spring.data.redis.host=localhost
```

#### 3. 生产环境建议（使用环境变量）
```properties
# MySQL配置
spring.datasource.url=${DB_URL:jdbc:mysql://localhost:3306/project_dev?useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true}
spring.datasource.username=${DB_USERNAME:remote_user}
spring.datasource.password=${DB_PASSWORD:Lzp1204@}

# Redis配置
spring.data.redis.host=${REDIS_HOST:localhost}
spring.data.redis.port=${REDIS_PORT:6379}
spring.data.redis.password=${REDIS_PASSWORD:Lzp1204@}
```

### 构建步骤
